# What is an LLMWiki

An LLMWiki is a knowledge base that an AI agent continuously maintains. You feed it raw material (notes, clippings, docs, half-formed ideas) and the agent organizes it into a living, interconnected set of articles that link to each other, stay deduplicated, and update as you add more. You read and chat with it the way you would a wiki, except you never hand-curate it.

## The problem it solves

The classic "second brain" promise is that capturing everything pays off later. In practice the capture is easy and the upkeep is not. Notes pile up, links rot, the same idea gets written three times in three places, and finding anything depends on you having tagged and filed it correctly months ago. The result is a graveyard of notes, not a knowledge base.

LLMWikis flip the labor. Capture stays cheap and messy. The synthesis, linking, and reorganization that used to require disciplined manual gardening becomes the agent's job. The knowledge base stays coherent without you tending it.

## Why they are popular

- AI assistants are now the default interface for retrieving and creating knowledge, so curating a base through the same chat interface feels natural rather than like learning another app.
- The synthesis is the part humans reliably skip. Offloading it removes the main reason second-brain systems decay.
- When the underlying store is plain markdown, you get the automation without lock-in. The files stay readable and portable even if the agent goes away.
- It scales the wrong direction from manual tools: the more you capture, the more useful it gets, instead of the more it rots.

## In short

A pile of documents becomes an encyclopedia that maintains itself. You bring the raw input and the questions. The agent handles the structure.
